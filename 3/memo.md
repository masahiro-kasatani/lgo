## Goでは配列が直接使われることは多くない
* 配列にはかなりの制限があるから
* Goでは配列の大きさを配列の「型」の一部としている
* スライスの後方支援のために存在している

## スライス
スライスの各要素はメモリ内の連続した領域に記憶されることで、素早い読み書きができるようになっています
各スライスはキャパシティをもっていて、あらかじめ一定個の連続する領域が確保される

## 宣言
```go
// 初期値がある場合
s := []int{1, 2, 3}
// サイズが分かる場合

// 分からない場合
s := make([]int, 0, 10)
```

## len & cap
```go
s := make([]int, 3, 10)
len(s)  // 3
cap(s)  //10
```
